The `Env` class is the fundamental building block for running embodied tasks. It connects three components — a **dataset** of episodes, a **simulator**, and a **task** — into a single training or evaluation loop. `RLEnv` wraps `Env` in a `gym.Env` interface for reinforcement learning. `VectorEnv` and `ThreadedVectorEnv` parallelize multiple environments for faster training.

## Env

`habitat.Env` is the primary single-process environment class. It loads the simulator and task from config, iterates through episodes, and exposes `reset`/`step` methods.

```python
from habitat import Env
from habitat.config import get_config

config = get_config("benchmark/nav/pointnav/pointnav_habitat_test.yaml")

with Env(config=config) as env:
    obs = env.reset()
    done = False
    while not done:
        action = env.action_space.sample()
        obs = env.step(action)
        done = env.episode_over
```

### Methods

#### `reset`

```python
def reset() -> Observations
```

Resets the environment and loads the next episode from the iterator. Clears the shortest-path cache of the previous episode, reconfigures the simulator, and calls `task.reset()`.

<ResponseField name="return" type="Observations">
  Initial observations from the simulator and task sensors after reset.
</ResponseField>

#### `step`

```python
def step(action: Union[int, str, Dict[str, Any]], **kwargs) -> Observations
```

Performs one action in the environment. Updates task measurements and step statistics. Raises an error if called before `reset()` or after the episode has ended.

<ParamField path="body.action" type="Union[int, str, Dict[str, Any]]" required>
  Action to perform. Can be a string action name, an integer index, or a dict with an `"action"` key and optional arguments (e.g., `{"action": "MOVE_FORWARD"}`).
</ParamField>

<ResponseField name="return" type="Observations">
  Observations after the action is applied.
</ResponseField>

#### `get_metrics`

```python
def get_metrics() -> Metrics
```

Returns all task measurements computed for the current step.

<ResponseField name="return" type="Metrics">
  A dict-like `Metrics` object containing metric names and their current values (e.g., `"distance_to_goal"`, `"success"`).
</ResponseField>

#### `seed`

```python
def seed(seed: int) -> None
```

Seeds the Python `random` module, NumPy, numba, the simulator, and the task.

<ParamField path="body.seed" type="int" required>
  Integer seed value.
</ParamField>

#### `reconfigure`

```python
def reconfigure(config: DictConfig) -> None
```

Reconfigures the simulator for the current episode. Called automatically inside `reset()`.

#### `render`

```python
def render(mode: str = "rgb") -> np.ndarray
```

Delegates to the simulator's render method.

<ParamField path="body.mode" type="str" default="rgb">
  Render mode string passed to the simulator.
</ParamField>

#### `close`

```python
def close() -> None
```

Shuts down the simulator. `Env` also supports the context manager protocol (`with Env(...) as env:`), which calls `close()` on exit.

## RLEnv

`habitat.RLEnv` wraps `Env` as a `gym.Env` subclass. Subclass it and implement the four abstract methods to define your reward function and termination condition.

```python
import habitat
from habitat import RLEnv

class MyRLEnv(RLEnv):
    def get_reward_range(self):
        return [-1.0, 1.0]

    def get_reward(self, observations):
        metrics = self.habitat_env.get_metrics()
        return metrics.get("distance_to_goal", 0.0) * -1.0

    def get_done(self, observations):
        return self.habitat_env.episode_over

    def get_info(self, observations):
        return self.habitat_env.get_metrics()

config = habitat.get_config("benchmark/nav/pointnav/pointnav_habitat_test.yaml")
with MyRLEnv(config=config) as env:
    obs = env.reset()
    obs, reward, done, info = env.step(env.action_space.sample())
```

</ParamField>

<ParamField path="body.dataset" type="Optional[Dataset]" default="None">
  Optional pre-constructed dataset passed directly to `Env`.
</ParamField>

</ResponseField>

</ResponseField>

<ResponseField name="episodes" type="List[Episode]">


<ParamField path="body.return_info" type="bool" default="False">
  If `True`, also returns `get_info(observations)` as the second element of a tuple.
</ParamField>

<ResponseField name="return" type="Union[Observations, Tuple[Observations, Dict]]">

</ResponseField>

#### `step`

```python
def step(*args, **kwargs) -> Tuple[Observations, Any, bool, dict]
```

Calls `habitat_env.step()`, then invokes `get_reward`, `get_done`, and `get_info`.

<ResponseField name="return" type="Tuple[Observations, Any, bool, dict]">

</ResponseField>

</ResponseField>

<ResponseField name="get_done(observations)" type="bool">

</ResponseField>

<ResponseField name="get_info(observations)" type="Dict[Any, Any]">
  Return an info dict for the current step (e.g., task metrics).
</ResponseField>

## VectorEnv

`habitat.VectorEnv` runs multiple environments in parallel worker **processes** using Python multiprocessing. It synchronises all environments on `step` and `reset`.

<Warning>
  `VectorEnv` uses `forkserver` by default which works well with CUDA. Avoid `fork` if you have already initialised CUDA on the main process.
</Warning>

```python
from habitat.core.vector_env import VectorEnv, _make_env_fn
from habitat.config import get_config

config = get_config("benchmark/nav/pointnav/pointnav_habitat_test.yaml")
num_envs = 4

with VectorEnv(
    make_env_fn=_make_env_fn,
    env_fn_args=tuple((config,) for _ in range(num_envs)),
) as envs:
    observations = envs.reset()
    actions = [envs.action_spaces[i].sample() for i in range(num_envs)]
    results = envs.step(actions)
```

### Constructor

```python
VectorEnv(
    make_env_fn: Callable[..., gym.Env],
    env_fn_args: Sequence[Tuple],
    auto_reset_done: bool = True,
    multiprocessing_start_method: str = "forkserver",
    workers_ignore_signals: bool = False,
)
```

<ParamField path="body.make_env_fn" type="Callable[..., gym.Env]" required>
  Factory function that receives the unpacked elements of each `env_fn_args` tuple and returns a `gym.Env` (typically `Env` or `RLEnv`).
</ParamField>

<ParamField path="body.env_fn_args" type="Sequence[Tuple]" required>
  One tuple of arguments per environment. `len(env_fn_args)` determines the number of parallel workers.
</ParamField>

<ParamField path="body.auto_reset_done" type="bool" default="true">
  Automatically call `reset()` on a worker as soon as its episode ends. Useful for seamless RL training loops.
</ParamField>

<ParamField path="body.multiprocessing_start_method" type="str" default="forkserver">
  How worker processes are spawned: `"forkserver"` (recommended with CUDA), `"spawn"`, or `"fork"`.
</ParamField>

<ParamField path="body.workers_ignore_signals" type="bool" default="false">
  When `True`, workers ignore SIGINT and SIGTERM and only exit when `close()` is called.
</ParamField>

### Properties

<ResponseField name="num_envs" type="int">

</ResponseField>

<ResponseField name="observation_spaces" type="List[spaces.Dict]">
  List of observation spaces, one per worker.
</ResponseField>

<ResponseField name="action_spaces" type="List[spaces.Dict]">
  List of action spaces, one per worker.
</ResponseField>

<ResponseField name="number_of_episodes" type="List[Optional[int]]">
  List of episode counts, one per worker.
</ResponseField>

## ThreadedVectorEnv

`habitat.ThreadedVectorEnv` is a drop-in replacement for `VectorEnv` that uses **threads** instead of processes. It is easier to debug (breakpoints work inside worker environments) but is slower for CPU-bound workloads due to the GIL.

```python
from habitat.core.vector_env import ThreadedVectorEnv, _make_env_fn
from habitat.config import get_config

config = get_config("benchmark/nav/pointnav/pointnav_habitat_test.yaml")

with ThreadedVectorEnv(
    make_env_fn=_make_env_fn,
    env_fn_args=tuple((config,) for _ in range(2)),
) as envs:
    print(envs.num_envs)  # 2
    observations = envs.reset()
```

<Tip>
  Use `ThreadedVectorEnv` when you need to set breakpoints inside the environment. Switch to `VectorEnv` for training runs.
</Tip>

`ThreadedVectorEnv` inherits all methods from `VectorEnv`. The only difference is the internal `_spawn_workers` implementation, which creates daemon `Thread` objects backed by `Queue` pairs instead of `Process` objects backed by `Pipe` connections.

## Agent

`habitat.Agent` is the abstract base class for any agent that acts inside `Env`. Subclass it and implement `reset()` and `act()` to create a benchmarkable agent.

```python
from habitat.core.agent import Agent
from habitat.core.simulator import Observations
from typing import Dict, Any, Union

class MyAgent(Agent):
    def reset(self) -> None:
        # Called at the start of each episode
        pass

    def act(self, observations: Observations) -> Union[int, str, Dict[str, Any]]:
        # Return an action based on current observations
        return "move_forward"
```

### Methods

#### `reset`

```python
def reset() -> None
```

Called before each new episode begins. Use it to reset any internal agent state (memory, counters, etc.).

#### `act`

```python
def act(observations: Observations) -> Union[int, str, Dict[str, Any]]
```

<ParamField path="body.observations" type="Observations" required>
  Dict-like object containing current sensor observations keyed by sensor UUID.
</ParamField>

<ResponseField name="return" type="Union[int, str, Dict[str, Any]]">
  Action to take. Can be an integer action index, a string action name (e.g. `"move_forward"`), or a dict with an `"action"` key and optional arguments.
</ResponseField>

## Benchmark

`habitat.Benchmark` provides a standardised evaluation loop for any `Agent` inside an `Env`. It aggregates per-episode metrics and returns the average over all evaluated episodes.

```python
from habitat.core.benchmark import Benchmark
from habitat.core.agent import Agent

class RandomAgent(Agent):
    def reset(self):
        pass

    def act(self, observations):
        import random
        return random.choice(["move_forward", "turn_left", "turn_right"])

benchmark = Benchmark(config_paths="benchmark/nav/pointnav/pointnav_habitat_test.yaml")
metrics = benchmark.evaluate(agent=RandomAgent(), num_episodes=10)
print(metrics)  # {"distance_to_goal": ..., "success": ..., "spl": ...}
```

### Constructor

```python
Benchmark(config_paths: Optional[str] = None, eval_remote: bool = False)
```

<ParamField path="body.config_paths" type="Optional[str]" default="None">
  Path to the Hydra config file used to create the `Env`. If `None`, the default config is used.
</ParamField>

<ParamField path="body.eval_remote" type="bool" default="false">
  If `True`, evaluation is run remotely through the EvalAI challenge server (requires challenge-specific dependencies). If `False`, evaluation runs locally.
</ParamField>

### Methods

#### `evaluate`

```python
def evaluate(agent: Agent, num_episodes: Optional[int] = None) -> Dict[str, float]
```

Runs the agent for `num_episodes` episodes (or all episodes in the dataset if `None`) and returns averaged metrics.

<ParamField path="body.agent" type="Agent" required>
  The agent to evaluate.
</ParamField>

<ParamField path="body.num_episodes" type="Optional[int]" default="None">
  Number of episodes to run. Must be ≤ the total number of episodes in the dataset.
</ParamField>

<ResponseField name="return" type="Dict[str, float]">
  Dictionary of metric names to their average values across all evaluated episodes (e.g. `{"distance_to_goal": 1.2, "success": 0.6, "spl": 0.55}`).
</ResponseField>

#### `local_evaluate`

```python
def local_evaluate(agent: Agent, num_episodes: Optional[int] = None) -> Dict[str, float]
```

Runs evaluation locally. Called internally by `evaluate()` when `eval_remote=False`.

#### `remote_evaluate`

```python
def remote_evaluate(agent: Agent) -> Dict[str, float]
```

Submits the agent to the EvalAI challenge server. Requires `evalai_environment_habitat` and gRPC dependencies installed in the challenge environment.
